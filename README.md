## ğŸ§  Security Concept

This application is built with a **security-first design philosophy**:

- Messages are **encrypted before transmission**
- Only authenticated peers can establish communication
- Encryption keys are never stored permanently
- Message content is accessible **only to the sender and the receiver**
- No third party can read or intercept message data

All communications occur in **private, isolated sessions**, ensuring confidentiality and integrity of exchanged messages.

---

## ğŸ”‘ Access Code Mechanism

The **Access Code** functions as a shared secret between users:

- Prevents unauthorized access
- Validates peer identity before connection
- Ensures both parties belong to the same secure session

Only users with the correct access code can communicate.
